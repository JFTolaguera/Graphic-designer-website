AboutWorkProcess

Project definition and contract

Via the form embedded in the about page, tell me the maximum about your project. Android design ? Web development ? The more info you give me, the more efficient it will run. Explain me your ideas, and what will be my part of the job.
Info I might need : Aim for this project ? Target audience, planning, competition ?
Then I draw a quotation, and send it to you by mail. If you accept the estimation, please sign and return the quotation to me.

Mock-ups, review and approval cycle

I'm a paper guy : I always draw some mockups on my sketchbook first, then design it on my graphic software/code editor, following your specifications. We set off a phase of reviewing, approving, fine tuning the mockup-ups until both you and me are satisfied by the design.
Main deliverables : written ideas, rough sketches, documented site map, wireframes, visual mock-ups, some lines of code for the basic design. For websites, I need all of the content provided by the client.

Development, creation

This is typically the main phase. After approval of the mock-ups, I design, code, create all the deliverables, meeting the deadlines. Regularly, I share a progress report and comments about the work. When work is almost finished, this is a time for a full review.

Delivery, payment and maintenance

I send you all drawables, icons or complete site structure with HTML, CSS and JS files for websites. If needed, I can make some polishing or small improvements. When everything is ok, I send you the invoice.
Payment by : PayPal, bank transfer (details on request) or cheque payable to Jean-François Tolaguera.
